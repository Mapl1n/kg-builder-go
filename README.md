# 🧠 档案知识图谱构建器

基于 **Go 语言纯规则引擎** 的中文命名实体识别（NER）与关系抽取系统。上传档案文本，自动提取人物、机构、合同并构建关联图谱，使用 **ECharts 力导向图** 可视化展示。

![Go](https://img.shields.io/badge/Go-1.23-00ADD8?logo=go)
[![License](https://img.shields.io/badge/license-MIT-green)](LICENSE)

## 💡 一句话说清

传统档案管理只能"看到一份文件"。知识图谱能让它**看到关联关系**——搜"张三"不仅返回他签的合同，还能看到他的关联公司、合作方、所有相关档案的完整网络。

## 🎯 核心特性

| 能力 | 实现 |
|------|------|
| 🧠 **中文 NER** | 纯正则 + 词典模式匹配，识别人物/机构/合同/日期/金额/地点 6 类实体 |
| 🔗 **关系抽取** | 法定代表人、签订合同、受雇于、合同金额、企业地址 5 类关系 |
| 🗄️ **图存储** | ElasticSearch 双索引（entity + relation），支持 BFS 遍历 |
| 📊 **ECharts 可视化** | 力导向图拖拽探索，6 色实体分类，节点聚焦展开关联 |
| 🔍 **模糊搜索** | 按名称/类型/来源文档搜索实体 |
| ⚡ **零依赖运行** | 纯 Go 内存图谱存储 + 本地 NER，无需 ES/Tika |

## 🚀 快速开始

### 零依赖模式

```bash
git clone https://github.com/Mapl1n/kg-builder-go.git
cd kg-builder-go
go run ./cmd/server
# 打开 http://localhost:8083
```

上传一段文本 → 自动提取实体和关系 → 力导向图可视化。

### 完整模式（PDF 解析）

```bash
docker compose up -d   # 启动 ES + Tika
go run ./cmd/server
```

## 📡 API 端点

```
POST /api/build             上传文本/PF → 提取实体+关系 → 构建图谱
GET  /api/search?name=张三  搜索实体（模糊匹配）
GET  /api/graph?entity_id=  获取图谱可视化数据（BFS 展开）
GET  /api/entity-types      实体类型配置（颜色/标签）
GET  /api/health            服务健康检查
```

## 🏗️ 架构

```
文本输入 → 规则引擎 NER
              ├── 人物: 正则匹配 "法定代表人：张三"
              ├── 机构: 正则匹配 "XX有限公司"
              ├── 合同: 正则匹配 "HT-2024-001"
              ├── 日期: 正则匹配 "2024年3月15日"
              └── 金额: 正则匹配 "人民币50万元"
              ↓
         关系抽取引擎
              ├── 张三 → 法定代表人 → 李四科技
              ├── 李四科技 → 签订 → HT-2024-001
              └── HT-2024-001 → 金额 → 50万元
              ↓
         图存储 (ES / 内存)
              ↓
     ECharts 力导向图 ← 浏览器拖拽探索
```

## 🔧 技术栈

| 组件 | 用途 | 降级方案 |
|------|------|---------|
| **Apache Tika** | PDF/DOCX 解析 | .txt 直接读取 |
| **ElasticSearch** | 图存储 + 模糊搜索 | 内存 map |
| **纯 Go 规则引擎** | NER + 关系抽取 | — (本身零依赖) |
| **ECharts** | 力导向图可视化 | — (浏览器 CDN) |

## 🎯 与飞度智能的关联

飞度核心业务是**智慧档案管理**——存储 PDF 并提供关键词搜索。知识图谱是其产品的**自然升维方向**：

- 传统产品：存文件 → 搜关键词 → 看到一份文件
- 知识图谱：存文件 → 提取实体 → 构建关联 → 搜"张三" → 看到所有相关档案 + 关联人物/公司/合同网络
